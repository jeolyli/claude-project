## 1. 产品概述

### 1.1 产品定位

**灵记**是一款面向个人用户的轻量级日常财务管理工具，帮助用户快速记录每一笔消费与收入，清晰掌握资金流向，培养健康的消费习惯。

> 核心价值主张：**3秒完成一笔记录，一眼看清消费全貌。**

### 1.2 产品愿景

让每个人都能轻松管理自己的财务，从"不清楚钱花在哪"到"对每一笔支出了然于胸"。

### 1.3 核心目标

- **记录快**：打开即记录，最小化操作路径，单次记录不超过 3 步操作
- **看得懂**：用直观的图表呈现消费结构，无需财务背景也能理解
- **用得久**：数据私有化存储，用户长期留存，形成个人财务档案
- **管得住**：用户独立账户，数据完全隔离，保障财务隐私安全（V1.0.1 新增）

### 1.4 目标用户

| 用户画像 | 特征描述 | 核心诉求 |
|----------|----------|----------|
| 职场新人 | 刚工作 1-3 年，收入有限，想搞清楚钱花在哪 | 快速记账、月度统计、消费预警 |
| 学生群体 | 生活费有限，需要精打细算 | 预算管理、分类统计、简单易用 |
| 自由职业者 | 收入不固定，需要兼顾收支两端 | 收支全景、趋势分析、分类明细 |
| 理财初学者 | 想养成记账习惯，但怕麻烦 | 极简操作、提醒机制、成就感反馈 |

### 3.2 功能详细说明

#### 3.2.0 用户认证（P0 — V1.0.1 新增）

**功能描述：** 用户通过用户名和密码创建账户，登录后享受完全独立的数据空间。所有财务数据与账户绑定，不同用户之间数据完全隔离。

**3.2.0.1 注册**

**操作流程：**
1. 进入应用，首先展示登录页面
2. 点击"没有账号？立即注册"跳转到注册页面
3. 输入用户名（2-20 个字符，支持中英文、数字、下划线）
4. 输入密码（6-20 个字符，支持字母、数字、特殊符号）
5. 再次输入密码确认
6. 点击"注册"按钮完成账户创建
7. 注册成功后自动登录，进入首页

**详细规则：**

| 规则项 | 说明 |
|--------|------|
| 用户名长度 | 2-20 个字符，中文算 1 个字符 |
| 用户名格式 | 支持中文、英文、数字、下划线，不允许纯数字 |
| 用户名唯一性 | 全局唯一，注册时实时检测是否已被占用 |
| 密码长度 | 6-20 个字符 |
| 密码格式 | 至少包含字母和数字两种类型 |
| 确认密码 | 必须与密码输入完全一致 |
| 密码存储 | 前端对密码进行 SHA-256 哈希后再传输和存储，明文密码不落地 |
| 自动登录 | 注册成功后自动登录，保存登录凭证（localStorage），7 天内免登录 |

**3.2.0.2 登录**

**操作流程：**
1. 进入应用，展示登录页面
2. 输入已注册的用户名
3. 输入密码
4. 点击"登录"按钮
5. 验证通过后进入首页

**详细规则：**

| 规则项 | 说明 |
|--------|------|
| 登录凭证 | 用户名 + 密码（SHA-256 哈希） |
| 错误提示 | 用户名或密码错误时统一提示"用户名或密码错误"，不具体区分（防枚举攻击） |
| 登录状态保持 | 登录成功后保存登录状态到 localStorage，默认保持 7 天 |
| 自动登录 | 7 天内再次打开应用自动跳过登录页，直接进入首页 |
| 登录过期 | 超过 7 天后需重新输入密码登录 |
| 错误次数限制 | 连续 5 次登录失败后锁定 15 分钟（前端限制，清除缓存可绕过，仅用于防暴力破解提示） |

**3.2.0.3 退出登录**

| 规则项 | 说明 |
|--------|------|
| 退出入口 | "我的"页面底部"退出登录"按钮 |
| 退出行为 | 清除本地登录凭证和缓存数据，跳转回登录页 |
| 二次确认 | 退出前弹窗确认："退出后需要重新登录，确定退出吗？" |
| 数据保留 | 退出不清除已同步的数据，下次登录后可恢复 |

**3.2.0.4 切换账号**

| 规则项 | 说明 |
|--------|------|
| 切换入口 | "我的"页面"切换账号"按钮 |
| 切换行为 | 退出当前账号并跳转到登录页 |
| 本地数据 | 切换账号时清除当前用户的本地缓存，防止数据混淆 |

#### 3.2.1 快速记账（P0 - MVP 核心）

**功能描述：** 用户可以在首页快速完成一笔消费/收入的记录。

**操作流程：**
1. 进入首页，焦点自动定位到金额输入框
2. 输入金额（支持小数点后两位）
3. 选择分类标签（默认展示常用分类）
4. 系统自动填充当前日期
5. 点击"保存"或按回车键完成记录

**详细规则：**

| 规则项 | 说明 |
|--------|------|
| 金额输入 | 仅支持数字和小数点，最多两位小数；最小 0.01，最大 99999999.99 |
| 分类选择 | 预设 8 个一级分类（餐饮、交通、购物、居住、娱乐、医疗、教育、其他），每个支持选择二级子分类 |
| 日期默认 | 默认为当天，用户可手动修改为历史日期 |
| 备注字段 | 可选填，最多 200 字 |
| 收支类型 | 默认"支出"，可切换为"收入" |
| 快捷模板 | 支持将常用记录保存为模板，一键复用 |
| 数据归属 | 每条记录关联当前登录用户 ID（V1.0.1 新增） |

**交互要点：**
- 金额输入框自动弹出数字键盘
- 分类区域横向滑动选择，高亮当前选中
- 保存后给予轻反馈（动画 + 震动），停留在首页可继续记录

**预设一级分类及图标：**

| 分类 | 图标 | 典型二级分类 |
|------|------|-------------|
| 🍽️ 餐饮 | 餐具 | 早午晚餐、零食、外卖、聚餐、饮品 |
| 🚌 交通 | 汽车 | 公交地铁、打车、加油、停车、共享单车 |
| 🛒 购物 | 购物袋 | 日用品、衣物、数码、家居、美妆 |
| 🏠 居住 | 房子 | 房租、水电、物业、维修、日用品 |
| 🎮 娱乐 | 游戏手柄 | 电影、游戏、旅行、运动、KTV |
| 💊 医疗 | 医药 | 门诊、药品、体检、牙科 |
| 📚 教育 | 书本 | 课程、书籍、考试、培训 |
| 📦 其他 | 更多 | 人情、快递、宠物、其他杂项 |

#### 3.2.2 预算看板（P0 - MVP 核心）

**功能描述：** 首页顶部展示本月预算使用情况，一目了然。

**展示内容：**
- 本月总支出金额（大字突出显示）
- 预算使用进度条（绿色 → 黄色 → 红色渐变）
- 剩余预算天数 / 剩余预算金额
- "今日已花"与"日均预算"对比

**状态规则：**

| 预算使用率 | 视觉状态 | 含义 |
|-----------|----------|------|
| 0% - 60% | 🟢 绿色 | 预算健康，保持节奏 |
| 60% - 85% | 🟡 黄色 | 注意控制，接近预算线 |
| 85% - 100% | 🟠 橙色 | 预算紧张，谨慎消费 |
| > 100% | 🔴 红色 | 已超预算，需要关注 |

#### 3.2.3 统计分析（P1）

**功能描述：** 多维度可视化展示消费数据，帮助用户理解消费结构。

**统计维度：**

| 维度 | 图表类型 | 说明 |
|------|----------|------|
| 分类占比 | 饼图 / 环形图 | 展示当月各分类消费金额及占比 |
| 消费趋势 | 折线图 | 展示近 7 天 / 30 天 / 近 12 月消费趋势 |
| 日历热力图 | 日历热力 | 日历视图展示每日消费强度（颜色深浅） |
| 分类排行 | 柱状图 | 各分类消费金额降序排列 |

**筛选器支持：**
- 时间范围：本周、本月、近 3 月、近 6 月、自定义区间
- 收支类型：全部 / 仅支出 / 仅收入
- 分类筛选：可选择部分分类进行对比

#### 3.2.4 记录列表（P1）

**功能描述：** 按时间倒序展示所有流水记录，支持搜索、筛选和编辑。

**核心能力：**

| 功能 | 说明 |
|------|------|
| 时间分组 | 按"今天/昨天/本周/本月/更早"分组折叠展示 |
| 模糊搜索 | 支持按金额、备注关键字、分类名称搜索 |
| 高级筛选 | 按时间范围 + 分类 + 金额区间组合筛选 |
| 批量操作 | 支持批量删除（需二次确认） |
| 记录编辑 | 点击单条记录进入编辑态，可修改所有字段 |
| 滑动删除 | 列表项左滑露出删除按钮 |

#### 3.2.5 分类管理（P2）

**功能描述：** 用户可以自定义消费分类体系，让记账更贴合个人生活。

**管理能力：**
- 新增/删除/编辑一级分类
- 新增/删除/编辑二级子分类
- 为分类选择图标（内置 100+ 图标库）
- 为分类设置颜色标识
- 分类排序（长按拖拽排序）
- 分类启用/禁用（不影响历史数据）
- 预设分类不可全部删除，至少保留 3 个

#### 3.2.6 数据管理（P2）

**功能描述：** 用户对数据的导入、导出和备份能力。

**能力清单：**

| 功能 | 说明 | 格式 |
|------|------|------|
| 数据导出 | 按时间范围导出流水记录 | CSV / Excel |
| 报表导出 | 导出统计分析报表（含图表截图） | PDF / 图片 |
| 数据导入 | 从外部文件导入历史流水（需符合模板格式） | CSV |
| 本地备份 | 手动触发完整数据备份 | JSON |
| 云同步 | 数据云端同步，多设备共享 | - |

### 4.3 数据模型（核心）

```

├── created_at: timestamp
├── last_login_at: timestamp
└── is_active: boolean

Transaction（流水记录）

├── icon: string
├── color: string

├── sort_order: number
├── is_preset: boolean
└── is_active: boolean

Budget（预算）
├── id: string
├── user_id: string (关联用户) ★新增
├── year: number
├── month: number
├── total_budget: number
├── category_budgets: [ { category_id, amount } ]
├── created_at: timestamp
└── updated_at: timestamp

RecurringBill（周期账单）
├── id: string
├── user_id: string (关联用户) ★新增
├── name: string
├── amount: number
├── category_id: string
├── cycle: 'daily' | 'weekly' | 'monthly' | 'yearly'
├── cycle_day: number
├── next_date: string
├── note: string
└── is_active: boolean
```

### 4.4 localStorage 存储方案（Web 版 V1.0.1）

由于当前 Web 版不依赖后端服务，所有数据存储在浏览器 localStorage 中：

```
localStorage 键名规范：
├── lingji_current_user    → { id, username, login_time, expires_at }
├── lingji_users           → { [id]: { id, username, password_hash, created_at } }
├── lingji_data_{user_id}  → {
│       transactions: Transaction[],
│       categories: Category[],
│       budgets: Budget[],
│       recurringBills: RecurringBill[]
│   }
```

> **数据隔离原则：** 每个用户的财务数据存储在独立的 `lingji_data_{user_id}` 键下，切换用户时只加载当前用户数据，确保不同用户之间数据完全隔离。

### 4.5 应用初始化流程（V1.0.1 新增）

```
应用启动
    │
    ▼
检查 localStorage 是否存在有效登录凭证
```

| App 安装包大小 | ≤ 50MB |

### 6.2 安全要求（V1.0.1 更新）

- 密码使用 SHA-256 哈希存储，明文密码不落盘
- 登录失败统一提示"用户名或密码错误"，不区分具体原因（防用户枚举）
- 连续 5 次登录失败触发 15 分钟冷却期
- 用户数据按 user_id 分区存储，多用户数据完全隔离
- 退出登录/切换账号时清除当前用户本地缓存
- 本地数据不包含密码明文，仅存储密码哈希
- 云同步数据需端到端加密传输（TLS 1.3）
- 不收集任何个人身份信息（PII），仅需用户名（无需手机号或邮箱）
- 数据导出时提醒用户文件包含敏感财务信息

### 6.3 可用性要求

- 支持深色模式 / 浅色模式
- 支持中英文切换
- 无障碍适配（字体缩放、屏幕阅读器）
- 核心功能（记账、看预算）在首屏可见，无需滚动
- 关键操作支持撤销（5 秒内可撤销删除）
- 登录注册页在 1 屏内完成，无需滚动
- 输入框支持密码可见/不可见切换

## 10. 风险与假设

### 10.1 核心假设

| 假设 | 验证方式 |
|------|----------|
| 用户愿意每天手动记账 | 观察 MVP 次日/7 日留存 |
| 预设 8 大分类能覆盖 80%+ 场景 | 统计"其他"分类使用率 |
| 预算看板能激励用户控制消费 | 对比有/无预算设置用户的月均消费变化 |
| 用户名+密码注册足够简洁，不会阻碍用户进入 | 观察注册转化率和注册页跳出率（V1.0.1 新增） |
| 用户接受 7 天自动登录过期策略 | 观察重新登录后的用户流失率（V1.0.1 新增） |

### 10.2 潜在风险

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 记账操作繁琐导致用户放弃 | 留存率低 | MVP 严格遵循 3 步内完成设计，上线后关注操作时长数据 |
| 用户忘记记账 | 数据不完整 | V2 规划记账提醒功能，设定每日固定时间推送 |
| 手动输入比自动抓取落后 | 竞品压力 | 聚焦"轻量极简"差异化，不与自动记账产品正面竞争 |
| 隐私安全引发信任问题 | 用户拒绝使用 | 优先本地存储，云同步作为可选项，数据加密策略透明公开 |
| 用户忘记密码无法找回 | 用户流失（V1.0.1 新增） | V1.3 紧急排期"忘记密码"功能，当前版本先在注册时引导用户记牢密码 |
| 密码哈希被窃取后暴力破解 | 数据泄露（V1.0.1 新增） | 使用 SHA-256 + salt 加盐哈希（后续版本升级为 bcrypt/PBKDF2） |

| 薄荷记账 | 社交记账、界面清新 | 社区内容杂乱、隐私存疑 |

> **灵记的差异化定位：** 在"极简操作"和"功能深度"之间找到平衡 —— 比鲨鱼记账更专业（预算 + 趋势），比随手记更轻量（3 秒记账）。
