## 1. 产品概述

### 1.1 产品定位

**灵记**是一款面向个人用户的轻量级日常财务管理工具，帮助用户快速记录每一笔消费与收入，清晰掌握资金流向，培养健康的消费习惯。

> 核心价值主张：**3秒完成一笔记录，一眼看清消费全貌。**

### 1.2 产品愿景

让每个人都能轻松管理自己的财务，从"不清楚钱花在哪"到"对每一笔支出了然于胸"。

### 1.3 核心目标

- **记录快**：打开即记录，最小化操作路径，单次记录不超过 3 步操作
- **看得懂**：用直观的图表呈现消费结构，无需财务背景也能理解
- **用得久**：数据私有化存储，用户长期留存，形成个人财务档案

### 1.4 目标用户

| 用户画像 | 特征描述 | 核心诉求 |
|----------|----------|----------|
| 职场新人 | 刚工作 1-3 年，收入有限，想搞清楚钱花在哪 | 快速记账、月度统计、消费预警 |
| 学生群体 | 生活费有限，需要精打细算 | 预算管理、分类统计、简单易用 |
| 自由职业者 | 收入不固定，需要兼顾收支两端 | 收支全景、趋势分析、分类明细 |
| 理财初学者 | 想养成记账习惯，但怕麻烦 | 极简操作、提醒机制、成就感反馈 |

### 3.2 功能详细说明

#### 3.2.1 快速记账（P0 - MVP 核心）

**功能描述：** 用户可以在首页快速完成一笔消费/收入的记录。

**操作流程：**
1. 进入首页，焦点自动定位到金额输入框
2. 输入金额（支持小数点后两位）
3. 选择分类标签（默认展示常用分类）
4. 系统自动填充当前日期
5. 点击"保存"或按回车键完成记录

**详细规则：**

| 规则项 | 说明 |
|--------|------|
| 金额输入 | 仅支持数字和小数点，最多两位小数；最小 0.01，最大 99999999.99 |
| 分类选择 | 预设 8 个一级分类（餐饮、交通、购物、居住、娱乐、医疗、教育、其他），每个支持选择二级子分类 |
| 日期默认 | 默认为当天，用户可手动修改为历史日期 |
| 备注字段 | 可选填，最多 200 字 |
| 收支类型 | 默认"支出"，可切换为"收入" |
| 快捷模板 | 支持将常用记录保存为模板，一键复用 |

**交互要点：**
- 金额输入框自动弹出数字键盘
- 分类区域横向滑动选择，高亮当前选中
- 保存后给予轻反馈（动画 + 震动），停留在首页可继续记录

**预设一级分类及图标：**

| 分类 | 图标 | 典型二级分类 |
|------|------|-------------|
| 🍽️ 餐饮 | 餐具 | 早午晚餐、零食、外卖、聚餐、饮品 |
| 🚌 交通 | 汽车 | 公交地铁、打车、加油、停车、共享单车 |
| 🛒 购物 | 购物袋 | 日用品、衣物、数码、家居、美妆 |
| 🏠 居住 | 房子 | 房租、水电、物业、维修、日用品 |
| 🎮 娱乐 | 游戏手柄 | 电影、游戏、旅行、运动、KTV |
| 💊 医疗 | 医药 | 门诊、药品、体检、牙科 |
| 📚 教育 | 书本 | 课程、书籍、考试、培训 |
| 📦 其他 | 更多 | 人情、快递、宠物、其他杂项 |

#### 3.2.2 预算看板（P0 - MVP 核心）

**功能描述：** 首页顶部展示本月预算使用情况，一目了然。

**展示内容：**
- 本月总支出金额（大字突出显示）
- 预算使用进度条（绿色 → 黄色 → 红色渐变）
- 剩余预算天数 / 剩余预算金额
- "今日已花"与"日均预算"对比

**状态规则：**

| 预算使用率 | 视觉状态 | 含义 |
|-----------|----------|------|
| 0% - 60% | 🟢 绿色 | 预算健康，保持节奏 |
| 60% - 85% | 🟡 黄色 | 注意控制，接近预算线 |
| 85% - 100% | 🟠 橙色 | 预算紧张，谨慎消费 |
| > 100% | 🔴 红色 | 已超预算，需要关注 |

#### 3.2.3 统计分析（P1）

**功能描述：** 多维度可视化展示消费数据，帮助用户理解消费结构。

**统计维度：**

| 维度 | 图表类型 | 说明 |
|------|----------|------|
| 分类占比 | 饼图 / 环形图 | 展示当月各分类消费金额及占比 |
| 消费趋势 | 折线图 | 展示近 7 天 / 30 天 / 近 12 月消费趋势 |
| 日历热力图 | 日历热力 | 日历视图展示每日消费强度（颜色深浅） |
| 分类排行 | 柱状图 | 各分类消费金额降序排列 |

**筛选器支持：**
- 时间范围：本周、本月、近 3 月、近 6 月、自定义区间
- 收支类型：全部 / 仅支出 / 仅收入
- 分类筛选：可选择部分分类进行对比

#### 3.2.4 记录列表（P1）

**功能描述：** 按时间倒序展示所有流水记录，支持搜索、筛选和编辑。

**核心能力：**

| 功能 | 说明 |
|------|------|
| 时间分组 | 按"今天/昨天/本周/本月/更早"分组折叠展示 |
| 模糊搜索 | 支持按金额、备注关键字、分类名称搜索 |
| 高级筛选 | 按时间范围 + 分类 + 金额区间组合筛选 |
| 批量操作 | 支持批量删除（需二次确认） |
| 记录编辑 | 点击单条记录进入编辑态，可修改所有字段 |
| 滑动删除 | 列表项左滑露出删除按钮 |

#### 3.2.5 分类管理（P2）

**功能描述：** 用户可以自定义消费分类体系，让记账更贴合个人生活。

**管理能力：**
- 新增/删除/编辑一级分类
- 新增/删除/编辑二级子分类
- 为分类选择图标（内置 100+ 图标库）
- 为分类设置颜色标识
- 分类排序（长按拖拽排序）
- 分类启用/禁用（不影响历史数据）
- 预设分类不可全部删除，至少保留 3 个

#### 3.2.6 数据管理（P2）

**功能描述：** 用户对数据的导入、导出和备份能力。

**能力清单：**

| 功能 | 说明 | 格式 |
|------|------|------|
| 数据导出 | 按时间范围导出流水记录 | CSV / Excel |
| 报表导出 | 导出统计分析报表（含图表截图） | PDF / 图片 |
| 数据导入 | 从外部文件导入历史流水（需符合模板格式） | CSV |
| 本地备份 | 手动触发完整数据备份 | JSON |
| 云同步 | 数据云端同步，多设备共享 | - |

```

│          前端（UI 层）            │
```

### 4.3 数据模型（核心）

```
Transaction（流水记录）
├── id: string (UUID)
├── type: 'expense' | 'income'
├── amount: number (decimal)
├── category_id: string
├── sub_category_id: string (nullable)
├── date: string (YYYY-MM-DD)
├── note: string (nullable, max 200)
├── is_recurring: boolean
├── recurring_id: string (nullable)
├── created_at: timestamp
└── updated_at: timestamp

Category（分类）
├── id: string
├── name: string
├── icon: string
├── color: string
├── type: 'expense' | 'income' | 'both'
├── parent_id: string (nullable)
├── sort_order: number
├── is_preset: boolean
└── is_active: boolean

Budget（预算）
├── id: string
├── year: number
├── month: number
├── total_budget: number
├── category_budgets: [ { category_id, amount } ]
├── created_at: timestamp
└── updated_at: timestamp

RecurringBill（周期账单）
├── id: string
├── name: string
├── amount: number
├── category_id: string
├── cycle: 'daily' | 'weekly' | 'monthly' | 'yearly'
├── cycle_day: number
├── next_date: string
├── note: string
└── is_active: boolean
```

## 5. 视觉设计规范（设计语言）

### 5.1 设计理念

> 🎀 **粉嫩可爱 · 温暖治愈** —— 让记账变成一件甜甜的小事。

产品整体走 **Soft Pink Kawaii** 风格，用柔和粉嫩的色调消除财务管理的严肃感，让用户在温暖治愈的氛围中轻松管理自己的小金库。圆形、圆角、毛绒质感是关键视觉元素。

### 5.5 圆角规则

| 元素类型 | 圆角值 | 说明 |
|----------|--------|------|
| 大卡片（预算看板、统计面板） | `20px` | 柔和圆润 |
| 普通卡片（账单列表项） | `16px` | 统一圆润感 |
| 按钮（主按钮、分类标签） | `24px` | 胶囊形，可爱感 |
| 输入框 | `12px` | 略圆润 |
| 弹窗/浮层 | `24px` | 突出柔和 |
| 图标容器 | `12px` | 小圆角 |

> 🔑 **核心原则：少用直角，多用圆角。** 所有可交互元素圆角 ≥ 12px，塑造"软软的"触感。

### 5.6 字体规范

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| H1 | `28px` | Bold(700) | 页面大标题、金额大字 |
| H2 | `22px` | Bold(700) | 板块标题 |
| H3 | `18px` | SemiBold(600) | 卡片标题 |
| Body | `15px` | Regular(400) | 正文、列表项名称 |
| Caption | `13px` | Regular(400) | 辅助说明、时间标签 |
| Small | `11px` | Medium(500) | 角标、微小提示 |

> 字体推荐：系统默认中文字体（PingFang SC / Noto Sans SC），数字金额使用 **Tabular Nums** 等宽特性确保对齐。

### 5.7 动效规范

| 场景 | 动效 | 时长 |
|------|------|------|
| 页面切换 | 右进左出滑动 | 250ms，ease-out |
| 记账保存 | 金额飞入 + 心跳动画 + 轻微震动 | 400ms |
| 按钮点击 | 缩放 95% → 回弹 100% | 150ms，spring |
| 列表项出现 | 从下淡入上浮 | 300ms，stagger 50ms |
| 数字变化 | 滚动递增/递减 | 400ms |
| 加载中 | 粉色跳动圆点（三个） | 循环 800ms |
| 删除操作 | 缩小 + 淡出 | 200ms |

> 🎀 **趣味点缀：** 完成记账后，金额旁边随机飘出一颗小爱心（❤️）、小星星（⭐）或小花瓣（🌸），持续 1.5 秒后消失。

### 5.8 阴影与层级

| 层级 | 阴影值 | 用途 |
|------|--------|------|
| 低（卡片） | `0 2px 8px rgba(255,133,162,0.10)` | 普通卡片浮起感 |
| 中（浮层） | `0 4px 16px rgba(255,133,162,0.15)` | 下拉菜单、弹窗 |
| 高（模态） | `0 8px 32px rgba(255,133,162,0.20)` | 全屏弹窗、确认对话框 |

> 阴影统一使用主色（粉）的 RGBA 值，比常规灰黑阴影更柔和温暖。

### 5.9 图标风格

- **类型：** 线性图标为主，填充图标用于选中态
- **描边宽度：** `1.5px`（比常规 2px 更轻盈）
- **风格：** 圆角端点（Round Cap + Round Join）
- **色彩：** 默认 Neutral-700，选中/激活使用 Primary-400
- **尺寸体系：** `16px`（小）、`20px`（中）、`24px`（大）、`32px`（特大）

## 6. 非功能需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 应用冷启动 | ≤ 2 秒 |
| 记账操作响应 | ≤ 200ms（保存后反馈） |
| 统计图表渲染 | ≤ 1 秒（1000 条数据量级） |
| 列表滚动帧率 | ≥ 55 FPS |
| 本地数据库查询 | ≤ 100ms |
| App 安装包大小 | ≤ 50MB |

### 6.2 安全要求

- 本地数据使用 SQLite 加密存储（SQLCipher）
- 云同步数据需端到端加密传输（TLS 1.3）
- 不收集任何个人身份信息（PII），仅需设备标识
- 数据导出时提醒用户文件包含敏感财务信息

### 6.3 可用性要求

- 支持深色模式 / 浅色模式
- 支持中英文切换
- 无障碍适配（字体缩放、屏幕阅读器）
- 核心功能（记账、看预算）在首屏可见，无需滚动
- 关键操作支持撤销（5 秒内可撤销删除）

| 平台 | 最低版本 |
|------|----------|

## 7. MVP 版本定义

### 7.1 MVP 划范围原则

> **做少但做精**：MVP 只包含"用户第一天就会用"的功能，让用户在 5 分钟内完成首次记账并看到价值。

### 7.2 MVP 包含功能

| 模块 | 功能 | 说明 |
|------|------|------|
| 快速记账 | 记录支出 | 金额 + 分类 + 日期，3 步完成 |
| 快速记账 | 记录收入 | 与支出同一入口，切换类型 |
| 预算看板 | 月度总预算 | 设置预算 → 实时进度条 |
| 预算看板 | 超支提示 | 超预算时首页变色提醒 |
| 账单列表 | 时间分组列表 | 查看历史记录 |
| 账单列表 | 基础搜索 | 按备注关键字搜索 |
| 统计分析 | 分类饼图 | 当月消费结构一目了然 |
| 统计分析 | 7 天趋势线 | 最近一周消费趋势 |
| 分类管理 | 使用预设分类 | 内置 8 大分类 + 子分类 |
| 数据管理 | 本地存储 | SQLite 本地持久化 |

### 7.3 MVP 不包含功能

- 自定义分类管理（先用预设）
- 云同步（V2）
- 周期账单（V2）
- 数据导出（V2）
- 快捷模板（V2）
- 日历热力图（V2）
- 多语言（先做中文）

## 9. 成功指标

### 9.1 北极星指标

> **周活跃记账天数**：用户每周至少有 5 天进行了记账操作。

### 9.2 核心指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 次日留存率 | ≥ 60% | 衡量首日体验质量 |
| 7 日留存率 | ≥ 40% | 衡量短期使用粘性 |
| 30 日留存率 | ≥ 25% | 衡量长期价值 |
| 日均记账笔数 | ≥ 3 笔 | 衡量使用深度 |
| 记账操作耗时 | ≤ 5 秒 | 从打开 App 到完成一笔记录 |
| 预算设置率 | ≥ 70% | 启用了预算功能的用户占比 |

## 10. 风险与假设

### 10.1 核心假设

| 假设 | 验证方式 |
|------|----------|
| 用户愿意每天手动记账 | 观察 MVP 次日/7 日留存 |
| 预设 8 大分类能覆盖 80%+ 场景 | 统计"其他"分类使用率 |
| 预算看板能激励用户控制消费 | 对比有/无预算设置用户的月均消费变化 |

### 10.2 潜在风险

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 记账操作繁琐导致用户放弃 | 留存率低 | MVP 严格遵循 3 步内完成设计，上线后关注操作时长数据 |
| 用户忘记记账 | 数据不完整 | V2 规划记账提醒功能，设定每日固定时间推送 |
| 手动输入比自动抓取落后 | 竞品压力 | 聚焦"轻量极简"差异化，不与自动记账产品正面竞争 |
| 隐私安全引发信任问题 | 用户拒绝使用 | 优先本地存储，云同步作为可选项，数据加密策略透明公开 |

### 11.1 竞品参考

| 产品 | 优势 | 劣势 |
|------|------|------|
