> 项目路径: `003.前端代码（前端工程师）/frontend/` + `003.前端代码（前端工程师）/backend/`  
> 版本: V1.0 | 日期: 2026-06-16

## 1. 项目概述

### 1.1 技术栈

| 子项目 | 技术 | 端口 |
|-------|------|------|
| **frontend/** | 纯原生 JS (ES Modules) + CSS3 + React 17 (login/register) + Chart.js | 文件直开 / Express 静态服务 |
| **backend/** | Node.js + Express 4 + mysql2 + bcryptjs + JWT | 3001 |

### 1.2 目录结构

```
frontend/
├── pages/          # 11 个 HTML 页面 (MPA)
├── js/             # 10 个 JS 模块 + vendor/
├── css/            # 11 个 CSS 文件
├── index.html      # 入口跳转
└── DESIGN.md       # 设计规范

backend/
├── server.js       # Express 入口
├── routes/auth.js  # 注册/登录 API
├── middleware/auth.js  # JWT 工具 (未实际使用)
├── db/connection.js   # MySQL 连接池
└── .env            # 环境配置
```

### 1.3 启动方式

```bash
# 后端
cd backend && npm run dev    # node --watch server.js (port 3001)

# 前端：直接打开 pages/*.html 或通过 Express 静态服务
```

## 3. 高风险功能识别

| 风险等级 | 功能 | 风险描述 |
|---------|------|---------|
| 🔴 高 | 后端同步 `pushDataToBackend` | 触发即忘(fire-and-forget)，无重试/队列，失败静默降级，数据可能丢失 |
| 🔴 高 | localStorage 为唯一持久化 | 清浏览器=全丢；JS模块 `storage.js` 无数据校验/版本迁移 |
| 🔴 高 | 密码哈希存于 localStorage | `lingji_users` 含 `password_hash`+`salt`，devtools可提取 |
| 🟡 中 | 头像 base64 存储 | 压缩至200x200但 base64 可能占满 localStorage 5MB 配额 |
| 🟡 中 | 双重后端混淆 | 前端调 `:8086` (Java) 但 Express 后端在 `:3001`，两套认证不互通 |
| 🟡 中 | JWT 被忽略 | 后端签发 JWT 但前端不存储/使用，认证仅靠 localStorage session |
| 🟡 中 | 隐私页 salt 为空 | `saveUser` 注册时 salt 为空字符串，密码验证可能失败 |
| 🟢 低 | Chart.js 动态加载 | stats.js 动态添加 script 标签，加载失败则图表空白 |

## 4. 测试分层方案

### 4.1 第一层：单元测试（JS 模块）

| 优先级 | 模块 | 测试重点 |
|-------|------|---------|
| P0 | `storage.js` | `getUserData/saveUserData` 读写一致性；`isSessionValid` 过期逻辑；`recordLoginFailure` 锁定/解锁 |
| P0 | `crypto.js` | `sha256` 已知向量验证；`hashPassword(password,salt)` 确定性；`generateSalt` 长度/字符集 |
| P1 | `budget.js` | `saveBudget` 输入验证(0/负/超大)；`updateCategorySum` 计算正确性 |
| P1 | `bills.js` | `getDateRange` 各时间段边界；`groupByTime` 分组正确性 |
| P2 | `categories.js` | CRUD 流程；名称去重检查；软删除限制(最少1个) |
| P2 | `data.js` | `exportCsv` BOM/转义；`backupJson/restoreFromJson` 格式验证 |

**工具**: Vitest + jsdom 或 Jest + jsdom

### 4.2 第二层：集成测试（Express API）

| 优先级 | API | 测试重点 |
|-------|-----|---------|
| P0 | `POST /api/auth/register` | 正常注册；重复用户名；密码为空；边界值 |
| P0 | `POST /api/auth/login` | 正常登录；错误密码；禁用用户；不存在用户 |
| P1 | `GET /api/health` | DB连接正常/断开两种状态 |
| P1 | 静态文件服务 | 各 HTML 页面可访问；SPA fallback |
| P2 | `authMiddleware` | JWT 签发/验证；过期token；篡改token |

**工具**: Supertest + Jest

### 4.3 第三层：端到端测试（页面流程）

| 优先级 | 流程 | 验证点 |
|-------|------|-------|
| P0 | 注册→登录→首页 | localStorage session 写入；自动跳转 |
| P0 | 首页快速记账 | 选择分类→输入金额→保存→预算进度更新→最近记录出现 |
| P1 | 账单筛选 | 搜索→类型/时间/分类筛选→结果正确→分组正确 |
| P1 | 预算设置 | 总预算→分类预算→差额计算→保存→后端sync日志 |
| P1 | 统计图表 | 有数据时 Chart.js 渲染折线图+圆环图；无数据时空状态 |
| P2 | 分类CRUD | 添加→编辑→删除→名称去重→最少1个限制 |
| P2 | 头像上传 | 选择图片→压缩→localStorage 存储→页面显示 |
| P2 | 密码修改 | 旧密码验证→新密码哈希→session刷新 |
| P2 | 数据导出/恢复 | CSV下载→JSON备份→从文件恢复→清除流水 |

**工具**: Playwright (chromium/firefox/webkit)

### 4.4 第四层：兼容性测试

| 平台 | 测试重点 |
|------|---------|
| Chrome (最新) | 全功能验证 |
| Safari (最新) | 全功能验证 |
| Firefox (最新) | Chart.js 兼容性 |
| 移动端 Chrome | 触屏交互、420px宽度适配 |
| 离线模式 | API不可用时 localStorage 降级能否正常工作 |

## 5. 测试数据准备

- **空用户**: 无交易/分类/预算的新注册用户
- **标准用户**: 含30天交易数据(支出+收入)、8个预设分类、已设置预算
- **边界用户**: 含999+条交易、超大金额(99999999)、超长备注(200字)
- **异常数据**: localStorage 数据被人为损坏(非JSON)、字段缺失
