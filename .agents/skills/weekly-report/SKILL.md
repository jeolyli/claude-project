# 周报生成器

从本地 git 提交记录自动生成工作周报。

## Step 1: 读取提交记录

```bash
# 获取本周 (周一至周日) 的所有提交
git log --since="last Monday" --pretty=format:"%h %ad %s" --date=short --author="$(git config user.name)"

# 如果本周提交为空，扩展到最近 7 天
git log --since="7 days ago" --pretty=format:"%h %ad %s" --date=short
```

## Step 2: 汇总分析

按以下分类梳理提交内容：

| 分类 | 匹配关键词 |
|------|-----------|
| **新功能开发** | 新增、feat、add、实现、创建 |
| **Bug 修复** | 修复、fix、bug、修正、解决 |
| **优化改进** | 优化、改进、重构、refactor、调整、更新 |
| **文档/配置** | 文档、README、配置、初始化、init |
| **测试** | 测试、test |

如果同一分类下有多条相关提交，合并描述。

## Step 3: 按模板生成周报

```markdown
# 工作周报

> 姓名: {从 git config user.name 获取}
> 周期: {本周一日期} ~ {本周日日期}
> 项目: 灵记 (LingJi) — 个人财务管理工具

## 本周工作概要

{一句话总结本周最重要的产出}

## 工作详情

### 1. {分类名}

- {提交描述1}
- {提交描述2}

### 2. {分类名}
...

## 下周计划

{如有从提交信息中可推断的后续工作}

## 遇到的问题 / 风险

{如有}
```

## Step 4: 保存

保存到 `000.文档纪要/周报_{本周一日期}_{本周日日期}.md`。

如果该文件已存在，覆盖写入。

## 注意事项

- 周报周期为本周一至本周日。如果当前是周日，包含当天；否则截止到昨天
- 如果本周无提交，生成一份空模板并标注"本周无代码提交"
- 提交描述保持原文，不做润色
- 不要编造工作内容
